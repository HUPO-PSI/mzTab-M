= mzTab-M Complete Column/Field Comparison: v2.0 vs v2.1
:toc: left
:toclevels: 3
:numbered:

Element-by-element comparison of all four mzTab-M sections (MTD, SML, SMF, SME) between version 2.0 and version 2.1.

*Legend:*

* 🟢 *Added in v2.1* – element is new, not present in v2.0
* 🔴 *Removed in v2.1* – element existed in v2.0 but is absent in v2.1
* ⚠️ *Changed* – one or more fields differ between versions
* Differing cell values are shown in *bold* in the detail rows

== Summary

[cols="4,1,1,1,1,1",options="header"]
|===
| Section | v2.0 elements | v2.1 elements | Added | Removed | Changed

| Metadata (MTD) Section | 67 | 75 | 10 | 2 | 65

| Small Molecule (SML) Section | 17 | 17 | 0 | 0 | 17

| Small Molecule Feature (SMF) Section | 12 | 12 | 0 | 0 | 12

| Small Molecule Evidence (SME) Section | 19 | 19 | 0 | 0 | 19

|===

| `ms_run[1-n]-usi_identifier`
a| An identifier for the MS run based on the Universal Spectrum Identifier (USI) specification. +
Implied within the USI is an MS run identifier. Every deposited MS run can be referenced with a shortened form: `mzspec:<collection>:<msRun>`. More info on the standard can be found: https://www.psidev.info/usi. +
Since an MS run may be represented in several formats (with potentially slightly different data associated with the spectrum), a format suffix MAY be specified (e.g. `mzspec:<collection>:<msRun>.RAW`) to signify that a specific file type is meant. +
If specified, the format suffix should align with the ms_run[1-n]-format parameter and the ms_run[1-n]-location suffix defined in the metadata section.
